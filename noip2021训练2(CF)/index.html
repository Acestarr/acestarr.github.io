CF1100E Andrew and Taxi

Description

给定一个有向图，改变其中某些边的方向，它将成为一个有向无环图。

现在求一个改变边方向的方案，使得所选边边权的最大值最小。

Solution

使得最大值最小，很明显二分答案。

对于二分的答案 xxx，考虑如何判断是否为有向无环图，也就是判断是否有环，可以 dfsdfsdfs 也可以 bfsbfsbfs，但是因为要构造一组解，所以这里使用拓扑排序。

只要能通过边权 >x>x>x 的边遍历每个点，就说明无环，同时将拓扑序记下来，用来求要改变哪些边。

在跑完拓扑后，将边权 ≤x\le x≤x 的边枚举一遍，如果 vvv 的拓扑序小于 uuu，也就是一条返祖边，就会产生环，需要改变方向。

CF632F Magic Matrix

Description

定义一个大小为 n×nn\times nn×n 矩阵 aaa 为魔法矩阵，当且仅当 aaa 满足以下条件:

以下 i,j,k∈Z+i,j,k\in \mathbb{Z^+}i,j,k∈Z+

  • ∀i∈[1,n],ai,i=0\forall i\in[1,n],a_{i,i}=0∀i∈[1,n],ai,i​=0

  • ∀i,j∈[1,n],ai,j=aj,i\forall i,j\in[1,n],a_{i,j}=a_{j,i}∀i,j∈[1,n],ai,j​=aj,i​

  • ∀i,j,k∈[1,n],ai,j≤max⁡(ai,k,aj,k)\forall i,j,k\in[1,n],a_{i,j}\le\max(a_{i,k},a_{j,k})∀i,j,k∈[1,n],ai,j​≤max(ai,k​,aj,k​)

给你一个矩阵，问它是不是魔法矩阵。 如果 aaa 是魔法矩阵输出 MAGIC\texttt{MAGIC}MAGIC，否则输出 NOT MAGIC\texttt{NOT MAGIC}NOT MAGIC。

1≤n≤2500,∀i,j∈[1,n],0≤ai,j≤1091\leq n\leq 2500,\forall i,j\in[1,n],0\le a_{i,j}\le 10^91≤n≤2500,∀i,j∈[1,n],0≤ai,j​≤109

第三条也就是对于任意 ai,ja_{i,j}ai,j​，在第 iii 行和第 jjj 行不存在同一列上的值都小于 ai,ja_{i,j}ai,j​。

Solution

第一条可以 O(n2)O(n^2)O(n2) 判断。

第二条可以 O(n)O(n)O(n) 判断。

考虑第三条

暴力的做法，对于 ai,ja_{i,j}ai,j​，枚举 kkk 判断 ai,j≤max(ai,k,aj,k)a_{i,j} \le max(a_{i,k},a_{j,k})ai,j​≤max(ai,k​,aj,k​)

但是这样是 O(n3)O(n^3)O(n3) 的，考虑优化

可以将每个点取出来，从小到大排序，然后一个一个模拟往回加，这样在当前数以前加进去的都是小于当前数的。

对于每行用 bitsetbitsetbitset 维护一下，初值为 000，若某个值为 111，说明是在前面加进去的，也就是小于当前数，判断时只需要将两行按位与，只要没有 111 就是合法的，然后将 (i,j)(i,j)(i,j) 改为 111。

注意对于值相同的数需要一起处理。

CF1131F Asya And Kittens

Description

有一个长为 nnn 的排列，一开始每个数都是一个独立的联通块。有 n−1n−1n−1 次操作，每次要求 xix_ixi​ 和 yiy_iyi​ 所在的联通块相邻，然后把这两个联通块合并。求一个合法的排列使得所有操作合法。保证有解。

1≤n≤1.5×1051 \le n \le 1.5\times 10^51≤n≤1.5×105

Solution

对于维护连通性的题，考虑并查集，但是要构造一个排列，该怎么办呢？

我们发现，对于每个操作，只需要两个联通块相邻即可，所以我们只需要维护每个联通快内的点，在合并时，将一个联通块内的点复制到另一个联通块后面就行了。

一开始每个联通块就是它自己。

为了保证时间复杂度，需要启发式合并，就是判一下两个联通块哪个小，将小的接在大的下面。

CF949C Data Center Maintenance

Description

给出 nnn 个数据中心，mmm 份资料。要把 mmm 份资料放到其中的两个数据中心备份，需要保证任意时刻都可以至少在一个数据中心进行备份。一天有 hhh 个小时，每个数据中心在一天内有一小时维护时间 uiu_iui​（0≤ui<h0 \leq u_i < h0≤ui​<h），在这一小时内该数据中心无法进行备份。

由于某种原因，需要把一些数据中心的维护时间向后推迟 1 小时（一个数据中心的维护时间的向后推迟可能导致有的资料无法在任意时刻进行备份 且 若推迟前 ui=h−1u_i = h - 1ui​=h−1 那么推迟后 ui=0u_i = 0ui​=0），请你求出最少需要向后推迟多少个数据中心，并把这些数据中心的编号输出出来。

2≤n≤105,1≤m≤105,2≤h≤1052\le n \le 10^5,1\le m \le 10^5,2\le h \le 10^52≤n≤105,1≤m≤105,2≤h≤105

Solution

对于第 iii 份资料，如果它的两个数据中心的维护时间相邻，那么在前面的推迟时，后面的也要推迟，就连一条从前面到后面边。

然后得到了一张图，不难发现推迟任意一个点后需要推迟的点的个数为这个点能到多少个点。

于是先 tarjantarjantarjan 缩点，得到一个 DAGDAGDAG，显然选出度为 000 的 sccsccscc 更优。

因此只需要枚举出度为 000 的 sccsccscc，取 sccsccscc 内点的个数的最小值即可。

CF85E Guard Towers

Description

已知 nnn 座塔的坐标，把他们分成两组，使得同组内的两座塔的曼哈顿距离的最大值最小。

在此前提下求出有多少种分组方案 mod  109+7\mod10^9+7mod109+7

n≤5000n\le 5000n≤5000

Solution

又是最大值最小，直接二分答案。

观察到是分成两组，并且组内边权 ≤x\le x≤x，两组间边权 >x>x>x，比较容易想到二分图。

对于二分的答案 xxx，只需要将 >x>x>x 的边连起来，进行01染色，判断是否为二分图。

考虑如何计算方案数，图不一定联通，对于每个联通块，是可以交换两边点的位置的，有两种方案，因此总方案数是 2cnt2^{cnt}2cnt，cntcntcnt 为联通块的个数。

那么联通块个数怎么求，其实就是在枚举每个点判断如果没被染色就进行01染色时，起点的个数。

CF711D Directed Roads

Description

有 nnn 个点和 nnn 条边，第 iii 条边从 iii 连到 aia_iai​ 。 每条边需要指定一个方向（可以将边反向）。问有多少种指定方向的方案使得图中不出现环。

2≤n≤2×105,1≤ai≤n,ai≠i2\le n \le 2\times 10^5,1\le a_i \le n,a_i \ne i2≤n≤2×105,1≤ai​≤n,ai​​=i

Solution

观察到 nnn 个点 nnn 条边，因此是个基环树森林。

设环上的边数为 tottottot，第 iii 个环上的边数为 cnticnt_icnti​，

对于环上的边，第 iii 个环上的方案数为 2cnti2^{cnt_i}2cnti​，但是有两种情况是环，因此是 ∏(2cnti−2)\prod (2^{cnt_i}-2)∏(2cnti​−2)。

对于不是环上的边，可以任意指定方向，方案数为 2n−tot2^{n-tot}2n−tot。

所以答案即为上面两种情况之积。

CF17E Palisection

Description

给定一个长度为 nnn 的小写字母串，问你有多少对相交的回文子串（包含也算相交）。

1≤n≤2×1061\le n \le 2 \times 10^61≤n≤2×106，将答案对 511239875112398751123987 取模。

Soluiton

正难则反，用总数减去不相交的。

考虑不相交的怎么算，用 lil_ili​ 表示以 iii 为左端点的回文串的个数，rir_iri​ 表示以 iii 为右端点的回文串的个数。

对于每个 lil_ili​ ，不相交的回文串的个数为 li×∑j=1irjl_i\times \sum_{j=1}^ir_jli​×∑j=1i​rj​，这样就不重不漏了。

但是 li,ril_i,r_ili​,ri​ 并不好求

先用 manachermanachermanacher 求出每个位置的最长回文半径，但是只有最长的，然而中间每个位置也都会 +1+1+1，所以用差分的小技巧，修改边界，最后跑一遍前缀和，就得到了 l,rl,rl,r 数组。

总数就是在所有回文串中任选两个的方案数。

注意最后在减去不相交的时候只枚举原串，不枚举在中间插入的字符。

CF40E Number Table

Description

有一个 nnn 行 mmm 列的棋盘，现在棋盘上的每个位置需要填上 111 或 −1-1−1 。 其中有 kkk 个位置已经填上了数，而其它的位置则是空的。 定义一个棋盘是好的当且仅当其每一行、每一列都满足所有数的乘积是 −1−1−1 。 你需要计算有多少种填数的方案是好的，答案对 ppp 取模。

1≤n,m≤103,0≤k<max(n,m),2≤p≤109+71\le n,m \le 10^3,0\le k < max(n,m),2\le p\le 10^9+71≤n,m≤103,0≤k<max(n,m),2≤p≤109+7

Solution

注意到 0≤k<max(n,m)0\le k < max(n,m)0≤k<max(n,m)，我们假设 n>mn>mn>m，如果 n<mn<mn<m，就交换行列。

这样，不论 kkk 个数怎么放，都会空出来一行，而空行可以让每一列乘积都成为 −1-1−1，就确定了，所以直接求前 n−1n-1n−1 行的方案数，也就不需要再考虑列了。

对于每一行，记一下已经填了几个数 cntcntcnt，和当前的乘积 mulmulmul，然后分类讨论：

  1. cnt=m,mul=1cnt=m,mul=1cnt=m,mul=1，无解
  2. cnt=m,mul=−1cnt=m,mul=-1cnt=m,mul=−1，直接跳过
  3. cnt=0cnt=0cnt=0 并且还没有选定空行，将这一行定为空行
  4. 否则，先去掉一个数用来将这一行的乘积调为 −1-1−1，剩下的就可以任意了， ans×2m−cnt−1ans \times 2^{m-cnt-1}ans×2m−cnt−1
