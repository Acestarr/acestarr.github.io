Problem 1. Lonely Photo

Description

给定一个长度为 nnn 的字符串 sss，sss 仅由 G 和 H 组成，求有多少个长度不小于 333 的子串只包含一个 G 或 H。

3≤n≤5×1053\le n \le 5\times 10^53≤n≤5×105

Solution

考虑对于每一个位置 iii，有多少个以 iii 结尾的合法的子串，不难发现子串的开头是一段连续的，所以只需要预处理一下 iii 前面第一个与 sis_isi​ 相同的位置 preipre_iprei​ 和第一个与 sis_isi​ 不同的位置 lstilst_ilsti​

然后只需要分类讨论一下以 iii 为结尾的子串中出现一次的字符是 G 还是 H。

  • si=si−1s_i=s_{i-1}si​=si−1​ 或 si−1≠si−2s_{i-1}\neq s_{i-2}si−1​​=si−2​ 时，也就是只出现一次的字符不是 sis_isi​，那么我们需要找到上一个与 sis_isi​ 不同的字符 sjs_jsj​，和 sjs_jsj​ 前面第一个与 sjs_jsj​ 相同的字符，中间都是合法的。
  • 否则，也就是只出现一次的字符是 sis_isi​，那么直接找到上一个与 sis_isi​ 相同的即可。

算是铜组最恶心的一道了吧，其实也不难就是细节比较多。

Problem 2. Air Cownditioning

Description

给定两个长为 nnn 的序列 a,ba,ba,b，每次操作可以将 aaa 序列中连续的一段就、 +1+1+1 或 −1-1−1，求最少需要操作多少次。

1≤n≤1051\le n \le 10^51≤n≤105

Solution

直接做差，然后就转化成典中典了

就是有正有负，通过手玩不难发现每次操作不可能同时操作正负，分成两边单独操作一样是最优的

所以只需要指针扫一遍找出每个正、负区间，然后跑一遍单调栈即可。

Problem 3. Walking Home

Description

给定一个 n×nn\times nn×n 的矩阵，. 可以通过，H 不可以通过，只能向右或向下走，求在至多转向 kkk 次的前提下从左上角走到右下角有多少种方案。

2≤n≤50,1≤k≤32\le n \le 50,1\le k\le 32≤n≤50,1≤k≤3

Solution

很容易想到状态

fi,j,k,df_{i,j,k,d}fi,j,k,d​ 表示从 (1,1)(1,1)(1,1) 走到 (i,j)(i,j)(i,j) 共转向 kkk 次，走过来后面向 d(0右,1下)d(0右,1下)d(0右,1下) 的方案数。

转移也比较简单，对于 d=0/1d=0/1d=0/1 分类讨论即可。

Problem 1. Closest Cow Wins

Description

在一条数轴上，有 kkk 个点有权值， pip_ipi​ 位置的权值为 tit_iti​，现在，对方已经占了 mmm 个位置 f1,f2,…fmf_1,f_2,\dots f_mf1​,f2​,…fm​，你要占 nnn 个位置（可以是小数），求最大的权值和为多少。

对于每个点，当这个点距离你占的最近位置小于对方占的最近位置时，你可以获得这个点的权值。

1≤k≤2×105,1≤m≤2×1051\le k \le 2\times 10^5,1\le m\le 2\times 10^51≤k≤2×105,1≤m≤2×105，所有这些 k+mk+mk+m 个位置均是 [0,109][0,10^9][0,109] 内的不同整数。

Solution

考虑对方占的每两个位置之间

当在这个区间内放 222 个点时，是可以得到这个区间内所有点的权值的。

所以只需要计算只放 111 个点时，最多能得到多少权值。

因为在这个区间内的点距离对面占的最近位置就是到两端的最小值，所以我们可以知道要想得到任意一个点的权值需要在哪个区间占位置

画一下图发现从左到右的点需要的区间的左右端点都是递增的，所以就可以双指针乱扫了（

我们有了在每个区间内占 1/21/21/2 个位置能得到的权值，接下来考虑如何占位置。

这里需要反悔贪心，先把占一个的权值都加到大根堆里，然后在被选时，将占两个的权值减去占一个的权值加到大根堆里，这样在下次选到时相当于占了两个。

Problem 2. Connecting Two Barns

Description

TTT 组数据，每组数据给定 nnn 个点 mmm 条无向边，现在至多建两条边，使得从 111 到 nnn 有路径，求建边的最小权值。

建一条从 iii 到 jjj 的边的权值为 (i−j)2(i-j)^2(i−j)2

1≤T≤20,1≤n≤105,∑(n+m)≤5×1051\le T \le 20,1\le n \le 10^5,\sum(n+m)\le 5\times 10^51≤T≤20,1≤n≤105,∑(n+m)≤5×105

Solution

先用并查集求出每个连通块，然后只需要求出 111 和 nnn 所在连通块到其他连通块的最短距离，枚举一遍求最小值即可。

在求最短距离时可以直接枚举每个点，在 111 和 nnn 所在连通块内二分找最小值

但我考场上没看见最后一个数据范围，就以为 O(Tnlog⁡n)O(Tn\log n)O(Tnlogn) 过不去，然后写双指针乱扫，结果还没二分快 qaq
